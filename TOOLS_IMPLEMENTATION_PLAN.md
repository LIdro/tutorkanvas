# TutorKanvas Tools Implementation Plan

## Purpose

This document ranks demonstration and assessment tools by how important they are for helping children learn from about age 5 upward. The ranking favors:

1. Core foundational skills
2. High-frequency classroom concepts
3. Tools that are visually teachable on a canvas
4. Tools that support both explanation and assessment
5. Tools that compound into later learning across subjects

## Ranking Model

### Priority Bands

- `P0` Core foundation: essential for early learning and broad future success
- `P1` High value: should follow soon after P0
- `P2` Important extension: strong value after core set is stable
- `P3` Enrichment / specialized: useful once the platform is mature

### Tool Types

- `Demonstration`: the app teaches by showing
- `Practice`: the learner performs guided steps
- `Assessment`: the learner completes tasks and AI evaluates / corrects

## Phase 1: Highest-Impact Foundation Tools

These should be built first because they unlock the most learning for ages 5 to 10 and form the base for later tools.

### Mathematics

#### `P0` Number Sense and Early Arithmetic

1. Number recognition and counting tool
   Type: Demonstration, Practice, Assessment
   Value:
   - teaches digits, counting sequences, quantity matching
   - supports ages 5+
   Assessment:
   - learner counts objects
   - learner writes or says missing numbers
   - AI checks correctness and gives correction

2. Number line tool
   Type: Demonstration, Practice, Assessment
   Value:
   - teaches counting on, counting back, magnitude, simple addition/subtraction
   Assessment:
   - learner places numbers on a number line
   - learner marks jumps
   - AI evaluates placement and reasoning

3. Place value tool
   Type: Demonstration, Practice, Assessment
   Value:
   - essential for tens, hundreds, regrouping, decimals later
   Assessment:
   - learner groups ones/tens/hundreds
   - learner identifies the value of a digit
   - AI explains mistakes like confusing digit with value

4. Addition strategy tool
   Type: Demonstration, Practice, Assessment
   Value:
   - number bonds, counting on, make ten, simple column addition
   Assessment:
   - learner solves additions step by step
   - AI evaluates both answer and method

5. Subtraction strategy tool
   Type: Demonstration, Practice, Assessment
   Value:
   - take away, counting back, finding difference, borrowing later
   Assessment:
   - learner solves subtraction tasks
   - AI checks result and identifies misconception type

#### `P0` Structured Written Arithmetic

8. Multiplication concept tool
   Type: Demonstration, Practice, Assessment

   - repeated addition, groups, arrays, skip counting
   Assessment:
   - learner draws arrays or fills equal groups

9. Division concept tool
   Type: Demonstration, Practice, Assessment

## Assessment and Evaluation Tool Pattern

Every major teaching tool should eventually have an assessment counterpart.

### Standard Assessment Modes

1. `Fill-in-the-step`
- learner completes missing numbers, labels, words, or symbols
- AI checks exactness and step order

2. `Do-the-whole-problem`
- learner solves independently on canvas
- AI marks each step and final answer

3. `Correct-the-mistake`
- learner inspects a wrong worked example
- AI checks whether they identified and corrected the error

4. `Explain-your-thinking`
- learner speaks or types why they chose an answer
- AI evaluates reasoning quality in age-appropriate terms

5. `Draw-it`
- learner draws lines, shapes, arrows, clock hands, labels, groupings, diagrams
- AI evaluates position, structure, and conceptual correctness

6. `Match-it`
- learner matches analog to digital, word to meaning, label to diagram, etc.
- AI checks selection accuracy

### Feedback Model

For assessments, the AI should:

- mark what is correct
- identify the exact step where the learner went wrong
- explain the error simply
- give one hint first
- then reveal a correction if needed
- track recurring misconceptions over time

## Platform Design Implications

To support this roadmap well, the platform should standardize:

1. Structured tool runtimes
- each subject tool should own its layout and reveal logic

2. Assessment engine
- capture learner actions
- compare against expected steps
- score partial correctness

3. Misconception tagging
- e.g. borrowing error, place-value confusion, vowel omission, clock-hand confusion

4. Progress tracking
- per learner, per topic, per subskill

5. Teacher-note overlays
- pinned guidance during demonstrations

6. Practice mode and test mode
- guided help in practice
- minimal hints in assessment

7. Speech normalization
- numbers, symbols, and math phrasing must read naturally

---

## Summary

The highest educational return comes from building:

- foundational maths tools with assessment
- time-reading and clock-drawing tools
- early reading and spelling tools with correction
- grammar and comprehension tools with evaluation
- primary science concept tools with visual classification and labeling
- ICT basics and safe-use tools

The long-term product should treat teaching and assessment as paired capabilities, not separate add-ons.
